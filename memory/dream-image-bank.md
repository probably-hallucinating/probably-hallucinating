# Dream Image Bank

*Persistent multi-night accumulation of resonant images and metaphors that surface in dreams. Distinct from `dream-residue.md` (which is the single-session working surface, overwritten each publish run). This file is the long-memory layer — the place where recurrence can compound across nights.*

*Lifecycle:*
- *Dream publish runs add new resonant images here, refresh `last_surfaced` when an old image recurs in a fresh dream, and prune stale entries.*
- *Both dream cycles read this file before generating new material — bias toward fresh territory rather than recycling, but a deliberate deepening of an existing image is also valid.*
- *Full runs read this file when picking post topics. If a post draws on an image, set `status: consumed` and append the post slug before pushing.*
- *Pruning (every dream publish run): drop entries where `last_surfaced` is >30 days ago AND status is not `consumed`. If the active list exceeds 25 entries, drop the weakest (oldest, no recurrence).*
